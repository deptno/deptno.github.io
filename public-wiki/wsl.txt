apt install python3-pip
pip install pipenv

<ul>
<li>하드웨어 리소스 설정이나 터널링등에 대한 설정이 <code class="span">.wslconfig</code> 파일을 통해 가능하다</li>
<li>탐색기에서 <code class="span">%userprofile%</code> 주소로 이동</li>
<li><code class="span">.wslconfig</code> 생성</li>
<li>기본값이나 설정할 수 있는 값을 확인하기 위해서는 관련 문서 참조 필요</li>
<li>CPU, memory 등을 통해서 하드웨어 스펙 전달 가능, <strong>재부팅</strong> 필요</li>
</ul>
<h2 id="외부에서-ssh-를-통해-접속">외부에서 ssh 를 통해 접속</h2>
<ul>
<li>외부라 함은 lan 을 의미</li>
<li>lan 은 <code class="span">192.168.x.x</code> 대역</li>
<li>wsl 은 <code class="span">172.x.x.x</code> 대역</li>
<li>이말인 즉슨<ul>
<li>A --ssh--&gt; B --nat--&gt; wsl</li>
<li>A --ssh--&gt; B --bridge-- wsl</li>
</ul>
</li>
<li>방화벽으로 <a href="ssh">ssh</a> 포트 개팡 필요 기본: <code class="span">22</code></li>
<li><a href="ssh">ssh</a> 최초 개방시 <code class="span">sshd_config</code> 에  <code class="span">PasswordAuthentication yes</code> 가 설정되어 있어야함</li>
</ul>
<h3 id="mirrorod">Mirrorod</h3>
<ul>
<li>windows host ip 를 공유</li>
<li>windows host 에 open ssh 가 떠있으면 wsl 까지 도달하지 못함</li>
</ul>
<h3 id="nat">NAT</h3>
<ul>
<li>windows 내에 자체 대역으로 ip 존재</li>
<li>windows host 에서 port-forwaring 필요</li>
</ul>
<h4 id="window-host-설정">window host 설정</h4>
<ol>
<li>window에서 openssh server 설치 및 시작</li>
<li>관리자 계정으로 power shell 오픈</li>
<li><code class="span">net user [ID] [PASSWORD] /add</code> 명령으로 사용자 생성, 패스워드 필요</li>
<li><code class="span">ssh [WINDOWS_HOST_IP]</code> 접속</li>
</ol>

## 설치

```sh
wsl --list --online # distro 목록

apt update

pip install pipenv
```

wsl --list --verbose # 설치 목록

- 하드웨어 리소스 설정이나 터널링등에 대한 설정이 `.wslconfig` 파일을 통해 가능하다
- 탐색기에서 `%userprofile%` 주소로 이동
- `.wslconfig` 생성
- 기본값이나 설정할 수 있는 값을 확인하기 위해서는 관련 문서 참조 필요
- CPU, memory 등을 통해서 하드웨어 스펙 전달 가능, **재부팅** 필요

## 외부에서 ssh 를 통해 접속
- 외부라 함은 lan 을 의미
- lan 은 `192.168.x.x` 대역
- wsl 은 `172.x.x.x` 대역
- 이말인 즉슨
  - A --ssh--> B --nat--> wsl
  - A --ssh--> B --bridge-- wsl
- 방화벽으로 [[ssh]] 포트 개팡 필요 기본: `22`
- [[ssh]] 최초 개방시 `sshd_config` 에  `PasswordAuthentication yes` 가 설정되어 있어야함

### Mirrorod
- windows host ip 를 공유
- windows host 에 open ssh 가 떠있으면 wsl 까지 도달하지 못함

### NAT
- windows 내에 자체 대역으로 ip 존재
- windows host 에서 port-forwaring 필요

#### window host 설정
1. window에서 openssh server 설치 및 시작
2. 관리자 계정으로 power shell 오픈
3. `net user [ID] [PASSWORD] /add` 명령으로 사용자 생성, 패스워드 필요
4. `ssh [WINDOWS_HOST_IP]` 접속

#### wsl os 설정
1. `openssh-server` 설치
```sh 
apt update
apt install openssh-server
```

#### remote -> window host 
> `net user` 를 통해 생성한 계정으로 접속
```sh 
ssh [USER]@[WINDOWS_HOST_IP]
```

#### remote -> window host -> wsl

### 종료
- [[ssh]] 로 접속한 상태에서 pc 자체를 종료할 경우
```sh
/mnt/c/Windows/System32/cmd.exe /c shutdown /s /t 0
```

## error
- wsl 실행하니 창 열리고 바로 꺼짐
- 지우고 설치하니 default profile, guid 어쩌고하고 꺼짐
- amd cpu 를 사용하고 있는데 bios 에서 svm 이 활성화가 꺼져있어서 생긴 문제로 다시 켜주면 실행됨
