개념만 확인하자</p></blockquote><h2 id="요약">요약</h2>
<ul>
<li>1 table === 1 application, model 이 아닌 application 과 매칭된다</li>
<li>테이블 생성시 <code class="span">hash key</code> 혹은 <code class="span">hash key</code> + <code class="span">range key</code> 로 unique key 조합을 선정<ul>
<li>아이템 입력시에는 반드시 <code class="span">unique key</code> 가 존재해야한다</li>
</ul>
</li>
<li>secondary index<ul>
<li>global secondary index<ul>
<li>table <code class="span">hash key</code>, <code class="span">range key</code> 에 대해서 모두 다른 컬럼을 선택해서 새롭게 인덱스를 생성한다</li>
</ul>
</li>
<li>local secondary index<ul>
<li>table <code class="span">hash key</code> 는 그대로 사용하고 <code class="span">range key</code> 만 다른 컬럼을 사용해서 인덱스를 생성한다</li>
</ul>
</li>
<li>index 로 사용되지 않을 컬럼은 schema 로 존재할 필요가 없으며 그냥 json 등의 컬럼에 넣어버리면된다<ul>
<li>검색용 컬럼이 분리된다는말</li>
<li>심지어 검색용 컬럼이 아닌 그냥 데이터는 바이너리로 압축해서 넣어도된다</li>
</ul>
</li>
</ul>
</li>
<li>scan 은 full scan  을 의미한다</li>
<li>query 는 인덱스를 선택하여 질의가 가능하며 몇가지 operator 가 제공된다</li>
<li>ttl 이 지원되어 특정 시점에 삭제를와 이벤트를 발생시켜 처리가 가능하다</li>
</ul>
