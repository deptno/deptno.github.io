<h2 id="code-signing">code signing</h2>
<blockquote><p>개발자 등록 후진행한다,  개발자 등록이 안되면 <a href="testflight">testflight</a> 를 포함한 배포단계가 동작하지 않는다</p></blockquote><ul>
<li>프로비저닝 프로파일은 다음을 포함있다<ul>
<li>개발자의 인증서(애플을 통해 인증되었기 때문에 폰에서 신뢰하기를 누르는 절차가 생략된다)</li>
<li>앱의 bundle id</li>
<li>device uuid</li>
</ul>
</li>
<li>기본적으로 개발자 인증서가 하나 발급되어있다. <a href="https://developer.apple.com" target="_blank">[URL] https://developer.apple.com/</a> 에서는 확인하지 못했으나 <em>xcworkspace</em> 를 열어서보면 확인된다<ul>
<li><em>Signing &amp; Capabiliteis</em> -&gt; <em>Signing</em> -&gt; <em>Team</em> 에 보면 추가되어있는 것을 확인할 수 있다</li>
<li>선택을 하게되면 <em>identifiers</em> 리스트에 추가된 것을 확인할 수 있다<ul>
<li><a href="https://developer.apple.com/account/resources/identifiers/list" target="_blank">[URL] https://developer.apple.com/account/resources/identifiers/list</a></li>
</ul>
</li>
<li>Product -&gt; Archive 해본다<ul>
<li><input checked="" disabled="" type="checkbox"> 이제 Validate 진행된다</li>
<li><input checked="" disabled="" type="checkbox"> Release Testing<ul>
<li><code class="span">xcarchive</code> 파일이 <code class="span">/Users/[USER_NAME]/Library/Developer/Xcode/Archives/[DATE]</code> 쪽에 저장</li>
<li>지정한 경로로 <code class="span">ipa</code> 파일 생성</li>
</ul>
</li>
<li><input disabled="" type="checkbox"> Testflight Internal Only</li>
<li><input disabled="" type="checkbox"> Testflight &amp; Appstore</li>
</ul>
</li>
<li><input disabled="" type="checkbox"> Product -&gt; Edit Schema -&gt; Release -&gt; Run 하게도면 디바이스 미등록 메시지가 나오고 등록을 눌러준다<ul>
<li><a href="https://developer.apple.com/account/resources/devices/list" target="_blank">[URL] https://developer.apple.com/account/resources/devices/list</a> 에 추가되는 것 확인</li>
<li>설치 실패</li>
<li><code class="span">Failed to install the app on the device.</code><ul>
<li><code class="span">Command Ld emitted errors but did not return a nonzero exit code to indicate failure</code></li>
<li><code class="span">IDERunOperationFailingWorker = IDEInstallCoreDeviceWorker;</code></li>
<li>Team 선택에서 앱스토어가 아닌 개발자 등록 이전에 쓰던 것으로는 사용가능</li>
<li>아마도 <strong>인증서가 바뀐경우 설치된 디바이스에서 수동으로 제거</strong> 가 필요한 것으로 추측<ul>
<li><a href="https://developer.apple.com/forums/thread/739711" target="_blank">[URL] https://developer.apple.com/forums/thread/739711</a> pod 지우고 다시 설치하라는 글도 있다</li>
</ul>
</li>
</ul>
</li>
<li><a href="testflight">testflight</a> 설정</li>
<li>일반 &gt; VPN 및 기기 관리 -&gt; VPN 쪽으로 이동한 것으로 추측됨</li>
</ul>
</li>
</ul>
</li>
</ul>

## code signing
> 개발자 등록 후진행한다,  개발자 등록이 안되면 [[testflight]] 를 포함한 배포단계가 동작하지 않는다

- 프로비저닝 프로파일은 다음을 포함있다
  - 개발자의 인증서(애플을 통해 인증되었기 때문에 폰에서 신뢰하기를 누르는 절차가 생략된다)
  - 앱의 bundle id
  - device uuid
- 기본적으로 개발자 인증서가 하나 발급되어있다. https://developer.apple.com 에서는 확인하지 못했으나 *xcworkspace* 를 열어서보면 확인된다
  - *Signing & Capabiliteis* -> *Signing* -> *Team* 에 보면 추가되어있는 것을 확인할 수 있다
  - 선택을 하게되면 *identifiers* 리스트에 추가된 것을 확인할 수 있다
    + https://developer.apple.com/account/resources/identifiers/list
  - Product -> Archive 해본다
    - [X] 이제 Validate 진행된다
    - [X] Release Testing
      - `xcarchive` 파일이 `/Users/[USER_NAME]/Library/Developer/Xcode/Archives/[DATE]` 쪽에 저장
      - 지정한 경로로 `ipa` 파일 생성
    - [ ] Testflight Internal Only
    - [ ] Testflight & Appstore
  - [ ] Product -> Edit Schema -> Release -> Run 하게도면 디바이스 미등록 메시지가 나오고 등록을 눌러준다
    - https://developer.apple.com/account/resources/devices/list 에 추가되는 것 확인
    - 설치 실패
    - `Failed to install the app on the device.`
      - `Command Ld emitted errors but did not return a nonzero exit code to indicate failure`
      - `IDERunOperationFailingWorker = IDEInstallCoreDeviceWorker;`
      - Team 선택에서 앱스토어가 아닌 개발자 등록 이전에 쓰던 것으로는 사용가능
      - 아마도 **인증서가 바뀐경우 설치된 디바이스에서 수동으로 제거** 가 필요한 것으로 추측
        + https://developer.apple.com/forums/thread/739711 pod 지우고 다시 설치하라는 글도 있다
    - [[testflight]] 설정
    - 일반 > VPN 및 기기 관리 -> VPN 쪽으로 이동한 것으로 추측됨
