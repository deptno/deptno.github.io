10:T932,<h1 id="upstage-ai-lab-day-19">upstage-ai-lab-day-19</h1>
<ul>
<li>4회차 제출</li>
<li>3일짜리 <a href="git">git</a> 강의</li>
<li>녹화는 rag</li>
</ul>
<h2 id="녹화-학습">녹화 학습</h2>
<ul>
<li>rag<ul>
<li>강사: 서머</li>
<li>수업에 모르는 용어가 너무 많아 수십초단위로 집중이 끊긴다</li>
<li>커리큘럼상 ml에 대해 아무것도 배우지 않은 상태에서 듣기가 어렵다고 생각되었다.</li>
<li>왜배우는지 모르고 배우는것들이 나열되어 있다.<ul>
<li>그럼에도 불구하고 논문 설명부터 시작한다</li>
</ul>
</li>
<li>이런한 부분들은 ml 에 대한 베이스가 있으신 분들의 경우 문제가 없을 수 있다.</li>
<li>수업을 다 들을 수 없는 수준이라고 생각해 패스하고 실습쪽에 코드만 리뷰하는 형태로 진행했다.</li>
<li>첫번째 프로젝트의 기반인데 개념자체는 어렵지 않기 때문에 진행에 어려움은 없을 것으로 생각된다</li>
<li>프로젝트는 langchain 이지만 수업에서는 같은 내용에 대해 <code class="span">llama-index</code>로의 구현도 포함되어있다.</li>
<li>강의 자체의 문제는 아닐 수 있고 내가 수업을 들을 수 있는 대상레벨이 아니었으리라고 생각된다.</li>
<li>rag 자체에 딥한 내용에 관심이 없기는 하지만 추후 ml에 대한 지식이 채워진 이후 다시들어보면 도움이 되지 않을까 싶다.</li>
<li>커리큘럼의 순서가 아쉽다</li>
</ul>
</li>
</ul>
<h2 id="라이브강의">라이브강의</h2>
<ul>
<li>최주호 강사의 git 수업이 있었다.</li>
<li><a href="git">git</a> 에 대해서는 잘 알고있기 때문에 간간히 진도 체크만 하고 내용을 듣진 않았다</li>
<li>다만 <a href="git">git</a> merge, <a href="git-rebase">git-rebase</a>, reflog, <a href="github">github</a> 까지 프로젝트에 필요한 부분을 충분히 커버하고 있다고 느꼈다.</li>
<li><a href="ssh">ssh</a> multi account 설정 같은 경우에는 고급주제인데 이 부분까지 수업에 존재했다.</li>
</ul>
