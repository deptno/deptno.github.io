11:T52b8,<h1 id="ruby">ruby</h1>
<h2 id="system-ruby">system ruby</h2>
<p>mac 을 설치하면 기본적으로 루비가 설치되어있다. </p>
<h2 id="support-archtecture">support archtecture</h2>
<pre><code class="hljs language-sh">$ ruby -v
ruby 2.6.8p205 (2021-07-07 revision 67951) [universal.x86_64-darwin21]
</code></pre><p><a href="universal">universal</a> 을 참고한다.</p>
<pre><code class="hljs language-sh">$ <span class="hljs-built_in">echo</span> <span class="hljs-string">&quot;ruby-3.1.0&quot;</span> &gt; .ruby-version
$ ruby -v
ruby 3.1.0p0 (2021-12-25 revision fb4df44d16) [arm64-darwin21]
</code></pre><p><a href="chruby">chruby</a> 를 사용하여 버전을 변경해보았따. 여기에는 <a href="arm64">arm64</a> 가 찍혀있다.</p>
<p>native 빌드가 당연히 성능적인 이점이 있겠지만,<br>관련된 <a href="gem">gem</a> 들이 모든 호환성이 보장되는지는 알 수 없다.</p>
<h2 id="ruby-gems-vs-bundler-정확하지-않음"><a href="ruby">ruby</a> gems vs <a href="bundler">bundler</a> <em>정확하지 않음</em></h2>
<p>루비는 <a href="ruby-gems">ruby-gems</a> 를 가지고 있지만 패키지 파일(<a href="path">Gemfile</a>, <a href="path">Gemfile.lock</a>)을 지원하지 않는다.</p>
<p>때문에 정확히 동일한 버저닝을 통해 여러 디펜던시를 관리하기 위해서는 이를 관리할 매니저가 필요한데 <a href="bundler">bundler</a> 가 이를 행한다.
<a href="bundler">bundler</a> 또한 하나의 <a href="gem">gem</a> 으로 버전을 가진다.</p>
<p>일단 <a href="bundler">bundler</a> 가 설치되면 이후부터는 <code class="span">bundle install</code> 을 통해 <a href="gem">gem</a> 들의 로컬(프로젝트 스코프) 설치가 가능해진다.
각 <a href="gem">gem</a> 들은 <code class="span">bundle exe GEM</code> 와 같은 형태로 사용이 가능하다.</p>

## system ruby
mac 을 설치하면 기본적으로 루비가 설치되어있다. 

## support archtecture
```sh
$ ruby -v
ruby 2.6.8p205 (2021-07-07 revision 67951) [universal.x86_64-darwin21]
```
[[universal]] 을 참고한다.

```sh
$ echo "ruby-3.1.0" > .ruby-version
$ ruby -v
ruby 3.1.0p0 (2021-12-25 revision fb4df44d16) [arm64-darwin21]
 ```
[[chruby]] 를 사용하여 버전을 변경해보았따. 여기에는 [[arm64]] 가 찍혀있다.

native 빌드가 당연히 성능적인 이점이 있겠지만,  
관련된 [[gem]] 들이 모든 호환성이 보장되는지는 알 수 없다.

## [[ruby-gems|ruby]] gems vs [[bundler]] *정확하지 않음*
루비는 [[ruby-gems]] 를 가지고 있지만 패키지 파일([[path|Gemfile]], [[path|Gemfile.lock]])을 지원하지 않는다.

때문에 정확히 동일한 버저닝을 통해 여러 디펜던시를 관리하기 위해서는 이를 관리할 매니저가 필요한데 [[bundler]] 가 이를 행한다.
[[bundler]] 또한 하나의 [[gem]] 으로 버전을 가진다.

일단 [[bundler]] 가 설치되면 이후부터는 `bundle install` 을 통해 [[gem]] 들의 로컬(프로젝트 스코프) 설치가 가능해진다.
각 [[gem]] 들은 `bundle exe GEM` 와 같은 형태로 사용이 가능하다.

## [[env]]
- `GEM_PATH`
- `GEM_HOME`

[[chruby]] 를 사용하여 버전을 변경해보았따. 여기에는 [[arm64]] 가 찍혀있다.

native 빌드가 당연히 성능적인 이점이 있겠지만,  
관련된 [[gem]] 들이 모든 호환성이 보장되는지는 알 수 없다.

## [[ruby-gems|ruby]] gems vs [[bundler]] *정확하지 않음*
루비는 [[ruby-gems]] 를 가지고 있지만 패키지 파일([[path|Gemfile]], [[path|Gemfile.lock]])을 지원하지 않는다.

때문에 정확히 동일한 버저닝을 통해 여러 디펜던시를 관리하기 위해서는 이를 관리할 매니저가 필요한데 [[bundler]] 가 이를 행한다.
[[bundler]] 또한 하나의 [[gem]] 으로 버전을 가진다.

일단 [[bundler]] 가 설치되면 이후부터는 `bundle install` 을 통해 [[gem]] 들의 로컬(프로젝트 스코프) 설치가 가능해진다.
각 [[gem]] 들은 `bundle exe GEM` 와 같은 형태로 사용이 가능하다.

## support archtecture
```sh
$ ruby -v
ruby 2.6.8p205 (2021-07-07 revision 67951) [universal.x86_64-darwin21]
```
[[universal]] 을 참고한다.

```sh
$ echo "ruby-3.1.0" > .ruby-version
$ ruby -v
ruby 3.1.0p0 (2021-12-25 revision fb4df44d16) [arm64-darwin21]
 ```
[[chruby]] 를 사용하여 버전을 변경해보았따. 여기에는 [[arm64]] 가 찍혀있다.

native 빌드가 당연히 성능적인 이점이 있겠지만,  
관련된 [[gem]] 들이 모든 호환성이 보장되는지는 알 수 없다.

## [[ruby-gems|ruby]] gems vs [[bundler]] *정확하지 않음*
루비는 [[ruby-gems]] 를 가지고 있지만 패키지 파일([[path|Gemfile]], [[path|Gemfile.lock]])을 지원하지 않는다.

때문에 정확히 동일한 버저닝을 통해 여러 디펜던시를 관리하기 위해서는 이를 관리할 매니저가 필요한데 [[bundler]] 가 이를 행한다.
[[bundler]] 또한 하나의 [[gem]] 으로 버전을 가진다.

일단 [[bundler]] 가 설치되면 이후부터는 `bundle install` 을 통해 [[gem]] 들의 로컬(프로젝트 스코프) 설치가 가능해진다.
각 [[gem]] 들은 `bundle exe GEM` 와 같은 형태로 사용이 가능하다.

- `GEM_HOME`
