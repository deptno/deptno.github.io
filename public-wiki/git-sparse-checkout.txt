10:T53c,<h1 id="git-sparse-checkout">git-sparse-checkout</h1>
<p>특정 위치만 <code class="span">checkout</code> 받는 기능이다.
루트 디렉토리의 파일들은 자동으로 받아진다.
특정 디렉토리를 지정할 수 있다.</p>
<ul>
<li><code class="span">--no-tags</code></li>
<li><code class="span">--depth 1</code></li>
</ul>
<p>위와 같은 옵션을 함께 사용하여 더 최적화된 데이터를 <code class="span">checkout</code> 할 수 있다.</p>
<ul>
<li><a href="https://git-scm.com/docs/git-sparse-checkout" target="_blank">[URL] https://git-scm.com/docs/git-sparse-checkout</a></li>
<li>CONE PATTERN SET</li>
</ul>
<h2 id="사용법">사용법</h2>
<pre><code class="hljs language-sh">git init
<span class="hljs-comment"># sparse checkout 기능을 enable 한다.</span>
<span class="hljs-comment"># --cone, CONE PATTERN SET 을 활성화한다.</span>
git sparse-checkout init --cone
<span class="hljs-comment"># 내려 받을 디렉토리를 선택한다.</span>
git sparse-checkout <span class="hljs-built_in">set</span> dir1 \
                        dir2 \
                        dir3/sub/path
git remote add origin git@github.com:USER/REPOSITORY.git
git fetch --no-tags --depth 1 origin BRANCH
git checkout --force BRNACH
</code></pre><h2 id="link">link</h2>
