10:T2139,<h1 id="vim">vim</h1>
<blockquote><p>현재(2023-10-08)는 <a href="neovim">neovim</a> 와 동이어가 되서 문서가 파편화 되었으니 둘다 참조가 필요하다
reference, back link 포함</p></blockquote><pre><code class="hljs language-vim"><span class="hljs-comment">&quot; ex command 결과보기</span>
:<span class="hljs-keyword">messages</span>
</code></pre><h2 id="font-설정"><a href="font">font</a> 설정</h2>
<h2 id="열기-옵션">열기 옵션</h2>
<pre><code class="hljs language-sh">$ vim -c <span class="hljs-string">&quot;24&quot;</span> file.txt
$ vim --clean <span class="hljs-string">&quot;24&quot;</span> file.txt
</code></pre><ul>
<li><code class="span">-c</code> : 첫 번째 파일이 열린 후 ex 명령어로 실행</li>
<li><code class="span">-R</code> : 읽기 전용으로 열기</li>
<li><code class="span">+[LINE_NUM]</code> : 해당 라인에서 열기</li>
<li><code class="span">+/[TEXT]</code> : 텍스트를 검색하여 열기</li>
<li><code class="span">-</code> : stdin 으로 입력을 받는 경우<code class="span">|</code> 사용으로 받는 경우등에 사용된다.</li>
<li><code class="span">-r</code> : 스왑파일 리스트 출력 <a href="vim-autoswap">vim-autoswap</a> 참고</li>
<li><code class="span">-O</code> : 여러 파일을 제공하면 창을 분할하여 보여준다. 수평/수직 분할(o/O)<pre><code class="hljs language-sh">$ vim -o $(git status -s | awk <span class="hljs-string">&#x27;{print $2}&#x27;</span>) <span class="hljs-comment"># 상태가 변한 애들 모두 연다.</span>
</code></pre></li>
<li><code class="span">--clean</code> : 플러그인 설정 로드 없이 연다.</li>
<li><code class="span">-M</code> : 수정 불가 전용으로 연다</li>
</ul>
<h2 id="registers">registers</h2>
<ul>
<li><p>i : 최근 input text</p>
</li>
<li><p>: : 최근 명령어</p>
</li>
<li><p>% : 현재 파일명</p>
</li>
<li><h1 id="-이전-버퍼명">: 이전 버퍼명</h1>
</li>
<li><p>[0-9] : 입력 모드에서 <code class="span">ctrl + r + [0-9]</code> 로 접근가능</p>
</li>
<li><p>[az] : 매크로에 사용</p>
</li>
<li><p>[AZ] : 매크로에 append 가 가능 <code class="span">:let @A=normal_command</code></p>
</li>
</ul>
<h2 id="ex-mode">ex-mode</h2>
<ul>
<li>move 0 : 가장 위로 이동</li>
<li>move +1 : 현재 라인 아래로 이동</li>
<li>move -1 : 현재 라인 위로 이동</li>
<li>copy 0 : 가장 위로 복사</li>
<li>copy +0 : 현재 라인 아래에 복사 yyp</li>
<li>copy +1 : 현재 라인 아래에 복사</li>
</ul>
<p><code class="span">copy</code> 는 <code class="span">t</code> 로 사용가능 <code class="span">move</code> 는 <code class="span">m</code> 로 사용가능.</p>

## 열기 옵션
```sh
$ vim -c "24" file.txt
$ vim --clean "24" file.txt
```
- `-c` : 첫 번째 파일이 열린 후 ex 명령어로 실행
- `-R` : 읽기 전용으로 열기
- `+[LINE_NUM]` : 해당 라인에서 열기
- `+/[TEXT]` : 텍스트를 검색하여 열기
- `-` : stdin 으로 입력을 받는 경우`|` 사용으로 받는 경우등에 사용된다.
- `-r` : 스왑파일 리스트 출력 [[vim-autoswap]] 참고
- `-O` : 여러 파일을 제공하면 창을 분할하여 보여준다. 수평/수직 분할(o/O)
  ```sh
  $ vim -o $(git status -s | awk '{print $2}') # 상태가 변한 애들 모두 연다.
  ```
- `--clean` : 플러그인 설정 로드 없이 연다.
- `-M` : 수정 불가 전용으로 연다

## registers
- i : 최근 input text
- : : 최근 명령어
- % : 현재 파일명
- # : 이전 버퍼명

- [0-9] : 입력 모드에서 `ctrl + r + [0-9]` 로 접근가능
- [az] : 매크로에 사용
- [AZ] : 매크로에 append 가 가능 `:let @A=normal_command`

## ex-mode
- move 0 : 가장 위로 이동
- move +1 : 현재 라인 아래로 이동
- move -1 : 현재 라인 위로 이동
- copy 0 : 가장 위로 복사
- copy +0 : 현재 라인 아래에 복사 yyp
- copy +1 : 현재 라인 아래에 복사

`copy` 는 `t` 로 사용가능 `move` 는 `m` 로 사용가능.

### vim <-> os
#### 파일명 복사
`+` 레지스터가 클립보드(mac)인걸 이용하여 아래와 같이 전달 한다.
```vim
" 현재 파일 경로
:let @+ = expand("%")
" 절대경로
:let @+ = expand("%:p")
" 디렉토리 까지
:let @+ = expand("%:h")
```

```vim
" 확인
:set clipboard?

:set clipboard^=unnamed
" append
:set clipboard+=unnamed
" override
:set clipboard=unnamed
```

## 패턴 검색
> `/` 로 시작

```vim
" 대소문자 구분 안함
:set ignorecase
" 대문자가 포함된 경우 구분 / 포함되지 않은 경우 구분 없이 검색
:set smartcase
```

```vim
# regexp
/\v[SEARCH_TEXT]
# 일반 문자열 검색
/\V[SEARCH_TEXT]
# 대소문자 구분 안함
/\c[SEARCH_TEXT]
# 대문자 구분
/\C[SEARCH_TEXT]
```

### 여러 파일 치환
- headless 모드에서 vim 을 [[sed]] 처럼 사용해서 치환
  ```sh
  vim **/*.md --headless --clean -c 'silent! argdo %s/\v(\`\`\`.*\n)##/\1\r##/g | update | q' 
  ```
  - `--headless` 는 vim 을 `background` 에서 실행 [[neovim]] 에서만 지원될 것으로 추정
  - `--clean` 은 치환처리에 필요없고 속도를 위해서 로드하지 않기 위해 추가
  - 모든 *markdown* 파일을 연 뒤에 `argdo` 로 열파일 모두에 대해서 치환 후 저장 그리고 종료

  + [[diary:2023-11-15]]

## 패턴 검색
> `/` 로 시작

```vim
" 대소문자 구분 안함
:set ignorecase
" 대문자가 포함된 경우 구분 / 포함되지 않은 경우 구분 없이 검색
:set smartcase
```

```vim
# regexp
/\v[SEARCH_TEXT]
# 일반 문자열 검색
/\V[SEARCH_TEXT]
# 대소문자 구분 안함
/\c[SEARCH_TEXT]
# 대문자 구분
/\C[SEARCH_TEXT]
```

### 여러 파일 치환
- headless 모드에서 vim 을 [[sed]] 처럼 사용해서 치환
  ```sh
  vim **/*.md --headless --clean -c 'silent! argdo %s/\v(\`\`\`.*\n)##/\1\r##/g | update | q' 
  ```
  - `--headless` 는 vim 을 `background` 에서 실행 [[neovim]] 에서만 지원될 것으로 추정
  - `--clean` 은 치환처리에 필요없고 속도를 위해서 로드하지 않기 위해 추가
  - 모든 *markdown* 파일을 연 뒤에 `argdo` 로 열파일 모두에 대해서 치환 후 저장 그리고 종료
  + [[diary:2023-12-19]]
  + [[diary:2023-11-15]]

## 열기 옵션
```sh
$ vim -c "24" file.txt
$ vim --clean "24" file.txt
```
- `-c` : 첫 번째 파일이 열린 후 ex 명령어로 실행
- `-R` : 읽기 전용으로 열기
- `+[LINE_NUM]` : 해당 라인에서 열기
- `+/[TEXT]` : 텍스트를 검색하여 열기
- `-` : stdin 으로 입력을 받는 경우`|` 사용으로 받는 경우등에 사용된다.
- `-r` : 스왑파일 리스트 출력 [[vim-autoswap]] 참고
- `-O` : 여러 파일을 제공하면 창을 분할하여 보여준다. 수평/수직 분할(o/O)
  ```sh
  $ vim -o $(git status -s | awk '{print $2}') # 상태가 변한 애들 모두 연다.
  ```
- `--clean` : 플러그인 설정 로드 없이 연다.
- `-M` : 수정 불가 전용으로 연다

## registers
- i : 최근 input text
- : : 최근 명령어
- % : 현재 파일명
- # : 이전 버퍼명

- [0-9] : 입력 모드에서 `ctrl + r + [0-9]` 로 접근가능
- [az] : 매크로에 사용
- [AZ] : 매크로에 append 가 가능 `:let @A=normal_command`

## ex-mode
- move 0 : 가장 위로 이동
- move +1 : 현재 라인 아래로 이동
- move -1 : 현재 라인 위로 이동
- copy 0 : 가장 위로 복사
- copy +0 : 현재 라인 아래에 복사 yyp
- copy +1 : 현재 라인 아래에 복사

`copy` 는 `t` 로 사용가능 `move` 는 `m` 로 사용가능.

### vim <-> os
#### 파일명 복사
`+` 레지스터가 클립보드(mac)인걸 이용하여 아래와 같이 전달 한다.
```vim
" 현재 파일 경로
:let @+ = expand("%")
" 절대경로
:let @+ = expand("%:p")
" 디렉토리 까지
:let @+ = expand("%:h")
```

```vim
" 확인
:set clipboard?

" override

```
