10:T789,<h1 id="bruno">bruno</h1>
<ul>
<li><a href="https://www.usebruno.com" target="_blank">[URL] https://www.usebruno.com/</a></li>
</ul>
<h2 id="feature">feature</h2>
<ul>
<li><a href="postman">postman</a> 대체제, api collection 을 자체 포맷인 <code class="span">.bru</code> 를 통해 저장한다.<ul>
<li>file 로 저장하기 때문에 <a href="git">git</a> 등 vcs 를 통한 공통 관리가 가능하다</li>
</ul>
</li>
<li>장황한 하다 주장하는 json 을 대신해 http 에 최적화된 포맷을 택했다.<ul>
<li>diff 등을 통해서 쉽게 파악 가능</li>
</ul>
</li>
<li>environments, secret 등을 지원, 변수화에 대한 것</li>
<li>pre, post scripting(<a href="javascript">javascript</a>) 을 지원하며 알만한 라이브러리(lodash, moment 등)을 빌트인으로 제공한다</li>
<li>자체 테스트를 지원한다</li>
</ul>
<h2 id="생각">생각</h2>
<ul>
<li>neovim 이 초기 extension 리스트업에서 빠진상태</li>
<li>기본적으로 gui 프로그램</li>
</ul>
<h3 id="장점">장점</h3>
<ul>
<li>백엔드 관점에서는 test를 지원해서 ci/cd 파이프라인에서는 편리하게 이용가능할 것으로 보임</li>
</ul>
<h3 id="단점">단점</h3>
<ul>
<li>초기버전인 <code class="span">.bru</code> 자체 포맷이 기존 플러그인들의 포맷대비 장점이 뚜렷하지않다<ul>
<li><a href="intellij">intellij</a> 의 자체 플러그인인 <code class="span">HTTP Client</code> 에도 비슷기능을 이미 지원하고있다.</li>
<li><a href="vim">vim</a> 쪽에도 유사한 플러그인이 오래전부터 존재했다.<ul>
<li><a href="https://github.com/diepm/vim-rest-console" target="_blank">[URL] https://github.com/diepm/vim-rest-console</a></li>
</ul>
</li>
</ul>
</li>
<li>프로젝트가 초기라는점</li>
</ul>

<ul>
